codemancy

[kohd-man-see]

Noun

1. ancient art of summoning complete code after just a few clicks

Context

- Hey, George, how did you learn to code PHP?

- Through Codemancy ^^


What can you do with it?

  • jump start

    Codemancy helps you start an application not with a seed, not with a doc, but with an actual complete boilerplate.

  • maintained

    Codemancy's Archwizards are maintaining the Factories so they are always up to date with the latest technologies. Some of them write those technologies, wink.

  • learn

    Did you always want to learn REST with PHP Lumen but got bogged down at step 3/452? Create an application using nothing but Codemancy and you get your own shiny application to explore.

  • don't learn

    Are you a Rank 99 Grandmaster in Front End Development, but a total noob on the Server Side? Don't worry, you can have a REST service up and running in no time, without writing a single line of code.

  • standardize

    Are you always copy-pasting work done by the grandfather of all company code? There's an easier way to make sure all microservices are written the same. Write a Codemancy Factory.

  • private

    Are you a more discrete Archwizard? You can stash your Factories privately, so only your friends can use them.

  • compose

    Factories can be combined and composed. That means you can stea... ahem borrow Factories written by other Archwizards, add some sugar on top and roll your own Factory.

  • brag

    Flash quiz: you have a fancy blog, with 99 tutorials on how to do various stuff. Each tutorial has a Git with the sample code. On average, for each subject, technology changes every 3 months. Do I have to ask the question? Yeah, you can roll a Factory and keep that updated instead.


So, how does it work?

There are 5 simple steps to get your application in a zip. Keep scrolling to see em'.
One
Step - 1 - Choose an application Factory

An application Factory is a Codemancy spell, created by our Archwizard community, to help you get started.

Something like Spring Boot with MySQL, or PHP Laravel, or NodeJs with Express and Mongo

Two
Step - 2 - Fill in the details

Like, what you want your application name to be, your author name on it (such as theGreatPatata, or evilCapuchin33).

Also, if you want OAuth2, or Domain Entites, or Swagger, or whatever else the Archwizard mixed into the Factory.

Three
Step - 3 - Download your application.

When the spell has finished brewing it, you can download your application as a zip archive.

Four
Step - 4 - ...
Nope, that's it. Three was the last step.
Five
Step - 5 - Seriously ...
You can start working on your application already. No more steps.
